## 5. 발견된 이슈

### ISSUE-01: 관리자 모바일 반응형 미지원 (심각도: HIGH)

- **현상:** 모바일 뷰포트(375px)에서 사이드바가 접히지 않고 항상 노출되어 메인 콘텐츠 영역이 심하게 압축됨
- **영향:** 모바일에서 관리자 페이지 사용이 사실상 불가능 (텍스트 잘림, 테이블 가독성 저하)
- **제안:**
  - (A) 모바일에서 사이드바를 기본 숨김 + 햄버거 메뉴 토글로 전환
  - (B) 반응형 breakpoint(768px 이하)에서 사이드바 오버레이 모드 적용
  - (C) 테이블에 가로 스크롤(`overflow-x: auto`) 적용

![ISSUE-01 스크린샷](../screenshots/admin_13_mobile_dashboard.png)

### ISSUE-02: Secret Key / API Key 평문 노출 (심각도: MEDIUM)

- **현상:** 간편로그인 설정, 결제(PG) 연동 설정에서 Client Secret, Secret Key 값이 마스킹 없이 평문으로 표시
- **영향:** 화면 공유, 스크린샷 유출 시 보안 키 노출 위험
- **제안:**
  - (A) Secret Key 필드에 `type="password"` 적용 + 눈 아이콘으로 표시/숨김 토글
  - (B) 저장된 키는 마스킹 표시 (예: `test_sk_****Bn14`)

![ISSUE-02 스크린샷](../screenshots/admin_11_payment_setting.png)

### ISSUE-03: 모달 ESC 키 닫기 미지원 (심각도: LOW)

- **현상:** 모달 오픈 상태에서 ESC 키를 눌러도 모달이 닫히지 않음
- **영향:** 키보드 사용자의 접근성(a11y) 저하
- **제안:** 모달 컴포넌트에 `keydown` 이벤트 리스너 추가하여 ESC 키 닫기 지원

### ISSUE-04: 모달 버튼 색상 불일치 (심각도: LOW)

- **현상:** 메뉴 등록 모달의 "저장 완료" 버튼이 시안(cyan) 색상이나, 배너/상품 모달의 저장 버튼은 기본 색상
- **영향:** 디자인 일관성 저하
- **제안:** 모든 모달의 저장/확인 버튼 색상을 통일

### ISSUE-05: 게시판 수정 UI 불일치 (심각도: LOW)

- **현상:** 배너/상품 관리는 모달 방식, 게시판 관리만 인라인 폼 방식으로 편집 UI가 상이
- **영향:** 관리자 경험의 일관성 저하
- **제안:** 게시판 수정도 모달 방식으로 통일하거나, 인라인 방식을 의도적 선택이라면 유지

### ISSUE-06: 대시보드 통계 데이터가 하드코딩 (심각도: LOW)

- **현상:** Overview 카드의 총 회원 수(1,248), 신규 게시글(45), 누적 트래픽(84,302)이 실제 데이터와 관계없이 고정값 표시
- **영향:** 관리자가 실제 현황을 파악할 수 없음
- **제안:** 백엔드 API(`/api/admin/stats`)와 연동하여 실시간 데이터 표시

## 7. 종합 평가

### 긍정적 요소
- 사이드바 네비게이션이 직관적이고 계층 구조(환경 설정 하위 메뉴)가 잘 구성됨
- 모든 관리 페이지에 CRUD 기능(등록/수정/삭제)이 일관되게 제공됨
- 다크 모드가 관리자 영역에서도 완벽하게 적용됨
- Active 메뉴 하이라이트로 현재 위치를 명확하게 표시
- 게시판 관리에서 다양한 옵션(비밀글, 댓글 등) 설정 가능
- 결제 PG사 3종(KCP, 카카오페이, 토스) 통합 관리 지원
- OAuth 5종(Naver, Google, Kakao, GitHub, Apple) 관리 지원
